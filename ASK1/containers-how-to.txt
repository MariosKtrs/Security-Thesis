Το πρώτο βήμα είναι να χτίσουμε έναν ευπαθή σέρβερ. Θα πρεπει να σηκώνουμε εναν σέρβερ που θα τρέχει κάποια εφαρμογή σε version που γνωρίζουμε ότι δεν είναι ασφαλής. 
Μια πολύ καλή επιλογή είναι το wordpress. Για να σηκώσουμε ένα wordpress, πρέπει να εγαταστήσουμε αναγκαία sql όπως και μια phpmyadmin για διαχείρηση των βάσεων.
Χρησιμοποιώντας docker containers μπορουμε να εγκαταστήσουμε όποιες version επιθυμούμε. Η προσέγγιση που ακολούθησα είναι η εξής:
Πρώτα φτιάχνω ένα dockerfile. Μέσα στο dockerfile κατεβάζω το base image απο το dockerhub, δηλαδή το wordpress.
Έπειτα στήνω το docker-compose.yml. Τo docker-compose.yml μας επιτρέπει να σηκώσουμε ταυτόχρονα και να συνδέσουμε μεταξύ τους όσα containers θέλουμε. 
Μέσα σε αυτό το αρχείο κατεβάζω τα δευτερεύοντα images (mysql,phpmyadmin,wpcli) που χρειάζονται για να έχουμε μια ολοκληρωμένη εφαρμογή και κάνω τις απαραίτητες
ρυθμίσεις για να έχουν το version που επιθυμώ και να επικοινωνούν μεταξύ τους. Σημαντικό είναι να κάνουμε και τα κατάλληλα health checks και sleep για να βεβαιωθουμε ότι
δεν θα φορτώσει το wordpress πριν την βάση ή ότι δεν θα εκτελεστεί το wp-cli πριν φορτώσει το wordpress. Το wp-cli χρησιμεύει στο να αυτοματοποιηθεί η διαδικασία
εγκατάστασης του wordpress (δημιουργία account κτλπ).
Αντί να φτιάχνει ο χρήστης καινούργιο λογαριασμό μόλις εγκαταστήσει το wordpress, μπορούμε με ένα σκριπτάκι που θα διαβάσει το wp-cli, 
να δώσουμε τις κατάλληλες παραμέτρους (username,password κλπ) για να γίνει αυτόματα η διαδικασία.

για το docker-compose, credits: https://github.com/vavkamil/dvwp

Υπάρχει το εξής πρόβλημα. Tο wordpress είναι σεταρισμένο να τρέχει στο localhost. Οπότε όταν θέλουμε να μπούμε στο wordpress μέσω του host machine, δουλεύει κανονικά.
Το πρόβλημα παρουσιάζεται όταν πάμε να το κάνουμε 
από άλλη συσκευή στο ίδιο δίκτυο. Μόλις πατήσουμε ip και port του υπολογιστή που τρέχει wordpress, 
μας δρομολογεί κατευθείαν στον localhost ο οποίος φυσικά δεν τρέχει τίποτα στον άλλο υπολογιστη. Για να διορθωθεί αυτό, κάνουμε log in στο wordpress, πηγαίνουμε
settings,general και αλλάζουμε το 127.0.0.1 που ειναι το localhost, 
σε μια static ip. Εναλλακτικά μπορούμε να αλλάξουμε το url parameter στο /bin/install.sh και να βάλουμε την ip που που θέλουμε να τρέχει το wordpress.

Για να ξεκινήσουμε, εκτελούμε "docker compose up" για να δημιουργηθούν τα container.
