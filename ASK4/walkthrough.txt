Σκοπός αυτής της άσκησης είναι η εκμάθηση της επίθεσης γνωστής και ως "directory traversal". Αυτή η επίθεση είναι εφικτή όταν ο προγραμματιστής
δεν εισάγει τους απαραίτητους ελέγχους για να εμποδίσει τον χρήστη από το να εισάγει κακόβουλο κώδικα σε παράμετρο που δέχεται το url μιας σελίδας.
Έτσι, μπορεί να μετακινηθεί εκτός του φακέλου της εφαρμογής και να έχει την ελευθερία να ανοίξει και να διαβάσει οποιοδήποτε αρχείο βρίσκεται στον υπολογιστή.

Για να χτίσουμε το container εκτελούμε "sudo docker image build -t vuln_php ."
Έπειτα το τρέχουμε με την εντολή "sudo docker run -p 8000:80 vuln_php"
Ανοίγουμε έναν browser της επιλογής μας και πληκτρολογούμε "localhost:8000"

Λύση:

Το πρώτο πράγμα που βλέπουμε είναι ότι έχουμε να κάνουμε με directory traversal attack. Σκοπός είναι να δρομολογηθούμε στο directory "home/secret/flag" και να ανοίξουμε
το αρχείο "flag.txt".
Κοιτάζοντας κάτω από την εικόνα βλέπουμε ένα warning για ένα missing array item που ονομάζεται file. Αυτό σημαίνει ότι το url παίρνει ως παράμετρο μια μεταβλητή
που λέγεται "file". Προσθέτωντας "?file=/home/secret/flag/flag.txt" στο url, παίρνουμε ένα μήνυμα λάθους για illegal characters. Αυτό μας λέει ότι η παράμετρος θέλει κωδικοποίηση.
Η τελεία αναπαρίσταται με %2e και το "/" με %2f 
Δοκιμάζοντας όμως την παράμετρο "?file=%2fhome%2fsecret%2fflag%2fflag%2etxt" παίρνουμε πάλι το ίδιο error.
Αυτό συμβαίνει επειδή το "%2f" και το "%2e" μεταφράζονται κυριολεκτικά σαν slash και τελεία αντίστοιχα. 
Για να ξεπεράσουμε αυτό το εμπόδιο, πρέπει να κωδικοποιήσουμε ξεχωριστά το %, το 2 και το e/f. 
Αυτό μεταφράζεται ως %25%32%46home%25%32%46secret%25%32%46flag%25%32%45txt 
Πληκτρολογώντας το, παίρνουμε τη σημαία μας.
